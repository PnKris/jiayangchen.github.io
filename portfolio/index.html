  • Distributed OTC Trading System

    2017.06
    • This trading system contains four modules which are Trader UI, Trader Gateway, Broker UI and Broker Gateway. This project is the implement of Broker Gateway using SSM framework. Broker_Gateway is designed for complex trading logic and it can support four kinds of orders included Market Order, Stop Order, Limit Order and Cancel Order. It can also maintain Market Depth for different products. It uses ActiveMQ queue mode to create queues and then receive orders from Trader_Gateway. It also uses topic mode to create queues to broadcast information to Trader User in the same company. It uses Redis to cache important orders' information and write back to database regularly. It also uses Websocket to push latest Market Depth to Broker UI. Nginx is used to implement load balance between Broker_Gateway.
